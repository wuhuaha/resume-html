# Security Policy

## 支持范围

当前项目以主分支为主要维护目标。示例内容和部署脚本仅作为参考，生产环境应按自己的安全要求加固。

## 敏感信息

不要提交以下内容：

- `backend/.env`
- API Key、后台密码、服务器登录凭据、GitHub Token
- 语音复刻参考音频
- 简历头像原图，除非你明确希望作为示例公开
- 未脱敏的个人隐私、客户信息或内部项目资料
- `storage/`、`output/`、前端构建产物和本地日志

## 生产部署建议

- 修改 `ADMIN_PASSWORD`，不要使用默认值。
- 使用 HTTPS，避免浏览器因不安全上下文禁用麦克风。
- 将后台管理能力放在可信网络、VPN、反向代理鉴权或更完整的登录系统之后。
- 只在可信环境中启用 `SHOWCASE_MODE=false` 的保存能力。
- 为第三方 AI/语音服务设置独立 API Key，并定期轮换。
- 对上传文件大小、类型和保存路径保持限制。
- 导出简历页脚中的 GitHub 地址、作者署名和项目说明应使用公开可访问且不含敏感参数的 URL。

## 漏洞反馈

如果你发现安全问题，请不要在公开 issue 中直接贴出可利用细节。建议通过私有渠道联系维护者，并提供：

- 影响范围。
- 复现步骤。
- 可能的修复建议。
- 是否涉及密钥、个人数据或远程代码执行。
